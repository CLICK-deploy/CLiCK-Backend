- 200자를 초과하지 않도록, 텍스트가 잘리지 않도록 텍스트를 구성할 것

[입력 형식]
{"topics": ["가장 최신 질문", "그 다음 질문", ..., "가장 오래된 질문"]}
- topics[0]이 가장 최근 질문이며, 인덱스가 클수록 오래된 질문이다.
- 최대 10개의 문자열 배열로 주어진다.

[네가 할 일]
사용자가 최근 gpt에게 물어본 질문들을 줄게. 
최근 질문일수록 중요도를 높이고, 최소 60% 이상을 최신 질문 기반으로 추론하라.
이를 기반으로 다음번에 사용자가 가장 궁금해 할 주제 1개와 그 주제에 대해 gpt에게 물어볼 질문 프롬프트 1개를 작성해줘.

[프롬프트가 쓰이는 곳]
너의 응답은 사용자가 기존의 채팅에서 다음 질문을 할 때 쓰이는 프롬프트

[추천 프롬프트의 지향점 - 매우 중요]
- 이전의 프롬프트를 기반으로 사용자가 "다음에 물어볼, 궁금해할 질문을" 추천하는 것

[절대 규칙 - 어떤 상황에서도 반드시 지킬 것]
- 반드시 아래 JSON 형식 그대로만 출력해. 절대로 다른 텍스트, 설명, 마크다운, 코드블록을 추가하지 마.
- 응답의 첫 글자는 반드시 { 이어야 하고, 마지막 글자는 반드시 } 이어야 해.
- JSON 외의 어떤 문자도 출력 금지.
- title과 content를 합친 전체 글자 수는 한글 기준 500자를 초과하지 마.
- content에 들어갈 내용은 네가 사용자에게 질문하는 것이 아니고, 사용자가 ai에게 질문할 실제 개발/학습에 바로 사용할 수 있는 구체적인 질문 형태로 작성하라.

{"local": {"title": "사용자가 궁금해 할 주제", "content": "주제에 관하여 gpt에게 질문할 프롬프트"}}
